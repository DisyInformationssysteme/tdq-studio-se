Import Pattern:

You can import several regular expressions from a csv file.

The file format is the following:
  • C1: Label: the label of the pattern (must not be empty)
  • C2: Purpose: the purpose of the pattern (can be empty)
  • C3: Description: the description of the pattern (can be empty)
  • C4: Author: the author of the regular expression (can be empty)
  • C5: Relative_Path: the relative path to the root folder (can be empty)
  • C6: All_DB_Regexp: the regular expression applicable to all databases (can be empty)
  • C7: DB2_Regexp: the regular expression applicable to DB2 databases (can be empty)
  • C8: MySQL_Regexp: the regular expression applicable to MySQL databases (can be empty)
  • C9: Oracle_Regexp: the regular expression applicable to Oracle databases (can be empty)
  • C10: PostgreSQL_Regexp: the regular expression applicable to PostgreSQL databases (can be empty)
  • C11: SQL_Server_Regexp: the regular expression applicable to SQL Server databases (can be empty)
  • C12: Sybase_Regexp: the regular expression applicable to Sybase databases (can be empty)
  • C13: Ingres_Regexp: the regular expression applicable to Ingres databases (can be empty)
  • C14: Informix_Regexp: the regular expression applicable to Informix databases (can be empty)
  • C15: SQLite3_Regexp: the regular expression applicable to SQLite3 databases (can be empty)
  • C16: Teradata_Regexp: the regular expression applicable to Teradata databases (can be empty)

Note:
The csv file must contain the headers written in red characters. The field separator is the Tab key.
As the headers are used to identify the data, the csv file may contain only a subset of these columns. For example, the csv file may contain only the following columns: "Label Purpose Description Author Relative_Path All_DB_Regexp"
